
Project_magazine
Песочница для симуляции бэкенда магазина: от генерации очередей клиентов и записей о покупках до резервного копирования серверной части и наблюдаемости. Есть лёгкий control plane: удалённая конфигурация сервисов, запуск/остановка процессов и запрос текущего состояния.
Состав и модули
* Base Project — генерация клиентов и касс для имитации случайных покупок; отправка отчётов в брокер сообщений.
* Cart Transit Interface — упрощённый брокер/транзит: временно сохраняет данные до подтверждения обработки и может отправлять запросы на разные реплики обработчика.
* Backup Server — хранит реплику БД; принимает и применяет дампы (full) и инкрементные бэкапы (prototype).
* Temporary Console UI — вспомогательная консоль (условный «бек-офис»): через общую библиотеку позволяет удалённо менять конфиги, запускать/перезапускать/останавливать отдельные приложения, запрашивать их текущее состояние; есть CRUD к БД.
* Api Transit Interface — главный API на EF Core, который управляет основной БД; основной трафик проходит через него.
Наблюдаемость и управление
* Control plane (custom): удалённая конфигурация, управление процессами, получение runtime-состояния сервисов.
* Logging & metrics (prototypes): самописные панели в стиле Kibana/Grafana для логов и простых метрик.
* Kafka-подобные очереди (custom) для упорядочивания обработки запросов.
Технологии
C#/.NET  ASP.NET Core Web API  EF Core (миграции)  MySQL/SQL Swagger/OpenAPI  GitHub Actions (CI) JWT Token
